# 🎵 Lyric Shadows

英語の歌詞から生まれた自然な会話でシャドーイング練習ができるWebアプリ。

## 機能

- YouTube サムネイル付き曲カード
- Gemini AI が歌詞をベースに自然な英会話を生成
- ElevenLabs で音声合成（シャドーイング用）
- 💬 会話タブ：文ごとに再生・日本語訳表示
- 🎵 Song タブ：歌詞（英語＋日本語）＋ YouTube 再生

## セットアップ

1. **依存関係インストール**
```bash
npm install
```

2. **`.env` ファイルを作成**
```
GEMINI_API_KEY=your_key
ELEVENLABS_API_KEY=your_key
YOUTUBE_API_KEY=your_key
PORT=3000
```

3. **サーバー起動**
```bash
node server.js
```

4. **曲を追加**
```bash
node add-song.js "曲名" "アーティスト名"
```

## 技術スタック

- **Backend**: Node.js / Express
- **AI**: Gemini 2.5 Flash（会話生成・歌詞翻訳）
- **TTS**: ElevenLabs
- **API**: YouTube Data API v3 / lyrics.ovh
- **Frontend**: Vanilla JS / CSS

## 📝 開発・運用ルール（ワークフロー）

本プロジェクトでは、今後の開発において以下の「パターンA」「パターンB」という呼称で機能を明確に切り分け、混同しないように進行します。

### パターンA（従来通り）: 曲 ➡️ 会話
*   **流れ**: 既存の楽曲・洋楽の歌詞をインプットとして、AIが自然な**日常会話（シャドーイングスクリプト）**を生成する。
*   **目的**: 好きな曲の世界観から英語を学び、シャドーイング練習に最適化する。（※現在のメイン機能）

### パターンB（新機能）: 会話 ➡️ 曲
*   **流れ**: ホテルでのチェックインなど、特定のシチュエーションの**日常会話（テーマ）**を入力として、AIが実用的な会話スクリプトを作成し、同時にそれを**Suno等の音楽生成AIでそのままコピペして曲にできる歌詞フォーマット**として出力する。
*   **絶対のルール⓪ ワークフロー（事前レビュー必須）**: 
    *   AIは【いきなり音声を生成してはいけない】。必ず先に「会話スクリプト」「Suno用歌詞」「Style」をチャット上でユーザーに提示し、ユーザーの【承認（OK）】または【修正の完了】を得てから初めてイレブンラボス等の音声生成・データ保存処理を実行すること。
*   **ルール① 品質（リアルな英語）**: わざとらしい安易（簡単・教科書的）な表現は使わず、ネイティブが普段のリアルなやり取りで使う「生きた英語（相槌やイディオム等）」を厳守する。
*   **ルール② ボリューム（ラリー回数）**: 会話の長さ・ラリー回数はパターンAと同じ規模感（約2分尺）を維持する。長すぎず短すぎず、音楽生成AIに一発で作らせるのに最適な分量とする。
*   **ルール③ 音声の役割分担（分離と自由度）**:
    *   **【💬 会話タブ】**：ElevenLabsによるTTS音声。実際のネイティブスピードでの実践的な「シャドーイング練習」用。
    *   **【🎵 Songタブ】**：SunoによるMP3音声。会話文の「重要フレーズ・生きた英語」は必ず盛り込むが、**一言一句まったく同じテキストにする必要はない**。歌としての心地よさ、キャッチーさ、オシャレさを優先して自由にメロディや構成を作って良い。
*   **絶対のルール④ Sunoフォーマットの提出必須項目**: 
    *   AIは、必ず【Sunoに入力するための音楽ジャンル指定（Style of Music）】を複数提案すること。絶対に出し忘れないこと。
    *   AIは、Sunoにコピペしやすくするため、【Suno用歌詞データ（日本語訳なし・英語のみバージョン）】を必ず別途出力すること。
    *   Sunoで生成した際に「1分で終わってしまう」「後半がずっと間奏になる」といった事態を防ぐため、AIは歌詞を必ず **[Verse 1] ➡️ [Chorus] ➡️ [Verse 2] ➡️ [Chorus] ➡️ [Bridge] ➡️ [Outro]** のような構成で出力し、「サビ（Chorus）」を反復させることで物理的な文字数を稼ぎ、およそ2分尺の正式な洋楽サイズになるようコントロールすること。
*   **ルール⑤ 出力のフォーマットと汎用性**:
    *   特定の個人名（Kazumaなど）を多用しすぎず、汎用的に使える名前や代名詞にとどめること。
    *   生成する `【会話タブ用】` のテキストは、発言者ごとに必ず空白行（1行あけ）を入れて視覚的に読みやすくすること。
    *   生成する `【Songタブ用】` の歌詞テキストも、Sunoへのコピペ時や人間が読むときに見やすいように、段落（セクション）ごとに適切に改行・空白行を入れて出力すること。
*   **ルール⑥ トップ画面のUI分離と専用表示**:
    *   アプリのホーム画面において、「パターンA（既存の洋楽）」と「パターンB（シチュエーション曲）」がごちゃ混ぜにならないよう、タブで表示を切り替えられる（フィルターできる）ようにする。
    *   パターンBの曲については、曲名だけでなく**「【ホテルの受付】」などのシチュエーション名（会話のテーマ）**がひと目で分かるようにカード（サムネイル）等に併記する。
